# 作問ガイド

ISUNARABE 合同演習2026 を作問するときに agent が ISUCON / ISUNARABE 特有の慣例・制約を
誤判断しないための知識を集約する。

## 構成と読む順序

1. **[norms.md](norms.md)**: ISUCON 競技として守られる規範 (追試レギュレーション、
   変更してよい/だめなものの境界、ベンチマーカーの典型挙動、スコア式の慣習)。実作業中の発見は
   随時追記する。
2. **[dev-loop.md](dev-loop.md)**: host 上で `cargo run` + bench を回すローカル開発ループ。
   webapp ロジック反復は VM を経由しないでこちらに分担する。

## 更新方針

- **norms.md**: 競技規範や ISUCON 慣習について新たな観察 (公式講評・新たなマニュアル条文・bench 設計
  事例) を得たら追記。
- **dev-loop.md**: ローカル開発手順や必要ツールが変わったら更新。
